Static vs. Dynamic Typing

March 9, 2016

Type

It is important to note here that typing in this case does not refer to the actual punching of keys on a computer keyboard, but rather refers to data type. In this case, static and dynamic are versions of typing--languages using static and dynamic typing are considered static typed or dynamic typed.

A well-documented difference between these types are the ways in which type checking is done, whether at compile-time or run-time. This results in the most important functional difference, which is the point at which variables are checked.

Static

Static-typed languages conduct variable checking at compile-time. This means that variables do not actually have to be defined before they are used. Since variables can be used without intitial definition, this means that static languages imply the need of explicit declaration of variables before they are utilized.

Examples of static-typed languages are C, C++ and Java.

Dynamic

Dynamic-typed languages conduct variable checking at run-time. This means that variables do have to be defined before use. Unlike static-typed languages, however, no explicit declaration of variables in necessary to begin utilizing variables.

Examples of dynamic-typed languages are Python, Ruby and PHP.

Which is better?

Neither, of course are inherently better than the other. A major pro of dynamic languages is the shorter code and the less time required to define and initialize variables. This however also leads to its major con, which is that without having to initially explicitly declare variables, it is very easy to make simple typing and misspelling errors and not realize it. Ultimately, it is contested by programmers and many promote one over the other, but it really comes down to personal preference: would you rather have the simplicity yet free realm of a dynamic language or the strictness yet overseeing structure of a static language?
