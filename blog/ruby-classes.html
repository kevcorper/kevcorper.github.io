Example of a Class

class fruit

attr_reader :name
attr_writer :price

def initialize
@name = name
@price = price
end

def display
puts "#{@name} costs $#{@price}"
end

end

Instance Variables
Instance Variables are defined using the symbol @ in front of the symbol name. This particular variable is used within defining a method with the intent of being able to cal upon that variable outside of the method. Without making the instance variable and instance, the variable would be considered local and would therefore only reside inside of the method. Using the @ symbol allows for the variable to exist outside of the method. This means that @name and @cost can be called outside of the method initialize (which is called whenever a new class object is created), as in the display method.

Attribute Readers/Writers
Atrribute readers and writers (written attr_reader and attr_writer) are used to allow the variables to be called outside of the method, similarly to instance variables, but in a different form. The attr method automatically makes the given variable a method as well. This means that in the above example, the attributes name and price can be called upon the class objects as methods. There is difference in how they can be called, however, since one is a reader and the other a writer. A reader attribute returns the value when called, but can not be modified. A writer does the opposite, allowing for the price to be modified but not returned outside of the class. Therefore, calling .name on a fruit object of the name "apple" would return "apple", while calling apple.price = 4.41 would change the apple price to 4.41.
